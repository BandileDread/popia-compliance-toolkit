## 3. OPERATOR OBLIGATIONS

### 3.1 Processing Instructions
The Operator SHALL:
- [ ] Process personal information ONLY on documented instructions from the Responsible Party
- [ ] Immediately inform the Responsible Party if instructed to process in violation of POPIA
- [ ] Refuse processing requests that violate POPIA or other applicable law

### 3.2 Confidentiality
The Operator SHALL:
- [ ] Ensure all personnel processing personal information are bound by confidentiality obligations
- [ ] Maintain confidentiality after employment/contract termination
- [ ] Not disclose personal information to unauthorized persons

### 3.3 Security Measures
The Operator SHALL implement appropriate technical and organizational measures:

| Security Domain | Minimum Requirements |
|-----------------|---------------------|
| **Access Control** | Unique user IDs, strong authentication, role-based access |
| **Encryption** | Encryption in transit (TLS 1.2+) and at rest (AES-256) |
| **Network Security** | Firewalls, intrusion detection, regular vulnerability scanning |
| **Physical Security** | Secure facilities, access logs, environmental controls |
| **Incident Response** | Detection, reporting, and response procedures |
| **Data Minimization** | Process only necessary data, regular purging |
| **Logging and Monitoring** | Audit trails of access and processing activities |

### 3.4 Sub-Processors

#### 3.4.1 Authorization
The Operator MAY engage Sub-Processors only:
- [ ] With prior written authorization from the Responsible Party
- [ ] Under a contract with equivalent data protection obligations

#### 3.4.2 List of Authorized Sub-Processors
Current authorized Sub-Processors:

| Name | Service | Location | Security Certification |
|------|---------|----------|----------------------|
| [Name] | [e.g., Cloud storage] | [Country] | [ISO 27001, SOC 2, etc.] |
| [Name] | [e.g., Backup] | [Country] | [Certification] |

#### 3.4.3 Notification
The Operator shall notify the Responsible Party of any changes to Sub-Processors at least [30] days in advance.

### 3.5 Data Subject Rights
The Operator SHALL:
- [ ] Assist the Responsible Party in responding to data subject requests
- [ ] Provide necessary information to fulfill access, correction, deletion requests
- [ ] Forward any direct requests from data subjects to the Responsible Party within [24] hours

### 3.6 Breach Notification
The Operator SHALL:
- [ ] Notify the Responsible Party of any personal information breach within [24] hours of discovery
- [ ] Provide details: nature, categories, approximate number of data subjects affected
- [ ] Cooperate in breach investigation and mitigation
- [ ] Not notify data subjects or regulators without Responsible Party consent (unless legally required)

## 7. RETURN AND DELETION OF DATA

### 7.1 Return Upon Termination
Upon termination or expiry of the underlying service agreement:

| Step | Timeline | Action |
|------|----------|--------|
| 1 | End of agreement | Operator stops all processing |
| 2 | [14 days] | Operator returns all personal information to Responsible Party |
| 3 | [14 days] | Operator deletes all copies (unless legally required to retain) |
| 4 | [21 days] | Operator provides written certification of deletion |

### 7.2 Exceptions
The Operator may retain personal information:
- [ ] To the extent required by applicable law
- [ ] For the establishment, exercise, or defense of legal claims
- [ ] With Responsible Party's written consent

### 7.3 Backup Data
The Operator shall ensure backup data containing personal information is:
- [ ] Returned or deleted according to agreed retention schedule
- [ ] Not retained beyond necessary period
- [ ] Securely erased using industry-standard methods
