# POPIA Compliance Checklist
## Protection of Personal Information Act (South Africa)

### Section 11 - Lawful Processing
- [ ] Documented legal basis for all personal information processing
- [ ] Consent mechanisms in place (where consent is the basis)
- [ ] Legitimate interest assessments completed (where applicable)
- [ ] Processing limited to specified, explicit, and lawful purposes

### Section 17 - Information Officer
- [ ] Information Officer appointed (default: CEO if not specified)
- [ ] IO registered with SAHRC (South African Human Rights Commission)
- [ ] IO responsibilities documented and communicated
- [ ] Deputy Information Officers appointed (if needed)

### Section 18 - Duties of Information Officer
- [ ] IO encourages compliance with POPIA
- [ ] IO deals with data subject requests
- [ ] IO works with Information Regulator
- [ ] IO ensures staff awareness through training

### Section 19 - Responsible Party Obligations
- [ ] Security measures implemented (technical and organizational)
- [ ] Breach notification procedures in place
- [ ] Data subject notification procedures for breaches
- [ ] Contracts with operators (processors) in place

### Section 21 - Security Measures
- [ ] Risk assessment conducted
- [ ] Security policy documented
- [ ] Encryption implemented (where appropriate)
- [ ] Access controls in place
- [ ] Regular security assessments scheduled

### Section 22 - Data Subject Participation (PAIA Integration)
- [ ] Procedure for access requests established
- [ ] Response timeframes defined (30 days)
- [ ] Fee structure for requests documented
- [ ] Correction/deletion procedures in place

### Section 23-24 - Purpose Specification
- [ ] Collection purposes explicitly defined
- [ ] Further processing compatible with original purpose
- [ ] Data minimization practiced
- [ ] Retention periods defined

### Section 69 - Direct Marketing
- [ ] Opt-in consent obtained for electronic marketing
- [ ] Opt-out mechanism in every communication
- [ ] Existing customer relationship documented (if using soft opt-in)
- [ ] Consent records maintained

### Section 72 - Cross-border Data Flows
- [ ] Cross-border transfers mapped
- [ ] Adequate protection in recipient country (or)
- [ ] Consent obtained from data subjects (or)
- [ ] Binding corporate rules / standard contractual clauses in place

### Documentation Required
- [ ] PAIA Manual (published)
- [ ] Privacy Policy (website)
- [ ] Processing Register
- [ ] Data Flow Diagrams
- [ ] Security Policies
- [ ] Incident Response Plan
- [ ] Training Records
- [ ] Data Subject Request Log
